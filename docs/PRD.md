## 1. 제품 개요

### 1.1 제품명
**Daily Scrum Timer** (데일리 스크럼 타이머)

### 1.2 목적
데일리 스크럼 미팅의 효율적인 시간 관리를 위한 웹 기반 타이머 애플리케이션

### 1.3 문제 정의
- 데일리 스크럼이 길어지는 경우가 빈번함
- 발표 순서를 정하는 데 시간 소요
- 휴가자를 매번 수동으로 체크해야 함
- 미팅 내용 기록이 번거로움

### 1.4 솔루션
- 팀원별 발표 시간을 제한하는 타이머 제공
- 랜덤 순서로 발표자 자동 선정
- 휴가자 간편 제외 기능
- (향후) 음성 인식 기반 자동 기록

---

## 2. 타겟 사용자

### 2.1 Primary User
- 스크럼 마스터 / 미팅 진행자
- 애자일 팀 리더

### 2.2 Secondary User
- 데일리 스크럼 참가 팀원

### 2.3 사용 환경
- 슬랙 허들 또는 화상회의 진행 중
- 화면 공유로 타이머를 팀원들에게 보여줌
- Chrome/Edge 브라우저 (권장)

## 5. 사용자 흐름 (User Flow)

### 5.1 기본 미팅 진행

```
[시작]
    ↓
[메인 화면] ─→ 팀원 추가/삭제 (메인에서 직접)
    ↓         휴가자 체크
[미팅 시작]
    ↓
[순서 셔플] ─→ 랜덤 순서 생성
    ↓
┌─[타이머 시작]
│     ↓
│ [카운트다운] ←─────────┐
│     ↓                 │
│ [10초 남음?]           │
│   │ Yes → 경고음       │
│   ↓                   │
│ [시간 종료?]           │
│   │ No ───────────────┘
│   ↓ Yes
│ [완료 알림 + 애니메이션]
│     ↓
│ [다음 사람] ─→ "Next: OOO" 미리 표시됨
│     ↓
└─[모든 팀원 완료?]
      │ No → 다음 팀원으로
      ↓ Yes
  [미팅 종료]
      ↓
  [결과 화면] ─→ (Phase 2) 슬랙 전송
                 (Phase 2) 녹음 기록 확인
```

│              ┌─────────────┐                    │

│           │                │    (색상 변화)      │

│           │                │                    │
│            ╲   ●───────────●                    │
│              └─────────────┘                    │
│                                                 │

│  🎤 현재: 홍길동                                 │

│  ─────────────────────────────────              │
│  ✅ 김철수                                       │

│  ⏳ 이영희                                       │

│  🏖️ 최수진 (휴가)                               │

### 6.2 원형 타이머 색상 변화

| 남은 시간 비율 | 색상 | 의미 |
|---------------|------|------|
| 100% ~ 50% | 🟢 녹색 (Green) | 여유 있음 |
| 50% ~ 20% | 🟡 노란색 (Yellow) | 주의 필요 |
| 20% ~ 0% | 🔴 빨간색 (Red) | 시간 부족 |

### 6.3 알림 타이밍

| 시점 | 알림 유형 |
|------|-----------|
| 10초 남음 | 경고음 (짧은 비프음) |
| 0초 (종료) | 완료 알림음 + 시각적 애니메이션 |

│                                                 │
│  ⏱️ 타이머 설정                                  │
│  ───────────────                                │
│  기본 시간:     [  1:00  ] ▼                    │
│  연장 단위:     [  30초  ] ▼                    │
│                                                 │
│  🔔 알림 설정                                    │

## 7. 기술 스택

### 7.1 MVP 기술 스택

| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 프레임워크 | Next.js 15 | SSR, 최신 React 지원 |
| 언어 | TypeScript | 타입 안정성 |
| UI | shadcn/ui | 커스터마이징, 접근성 |
| 스타일링 | Tailwind CSS | 빠른 개발, 일관성 |
| 애니메이션 | Motion | 부드러운 UX |
| 오디오 | use-sound | React 최적화 |
| **상태관리** | **Zustand** | 간단, 가벼움, persist 미들웨어 |
| 저장소 | localStorage (via Zustand persist) | 서버 불필요 |
| 배포 | Vercel | Next.js 최적화 |

## 8. MVP 범위 및 우선순위

### 8.1 MVP 포함 (Must Have)

1. **팀원 관리 (메인 화면)**
   - 팀원 추가/삭제
   - 휴가자 체크
   - 로컬 저장

2. **원형 타이머**
   - 원형 프로그레스 바
   - 중앙 숫자 표시
   - 색상 변화 (녹색→노란색→빨간색)
   - 시작/정지/리셋
   - +30초 연장
   - 다음 사람

3. **알림 시스템**
   - 10초 경고음
   - 종료 알림음
   - 시각적 효과

4. **발표자 표시**
   - 현재 발표자 강조
   - **다음 발표자 미리보기**
   - 완료 상태 표시
   - 랜덤 셔플

5. **테마**
   - 다크모드 토글
   - 테마 저장

6. **설정**
   - 기본 시간 조정
   - 소리 On/Off

### 8.2 MVP 제외 (Nice to Have / Future)

- 드래그앤드롭 정렬
- 다시 섞기 버튼
- 슬랙 연동
- 음성 인식
- AI 요약
- 화면 공유
- 펜 드로잉
